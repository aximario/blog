JS 中的原型链

作者：阿希

更新于：2018-12-17

不用颜色的箭头分别代表以下含义：

  • __proto__
  • prototype
  • constructor

JS 创造了第一个对象，然后用这个对象衍生出了各种各样的对象，有种道生一，一生二，二生三，三生万物的意思。而这第一个对象就是 Object.prototype，意思就是对象的原型，所有的对象都由这个原型演变而来。这就奠定了一个基础，JS 世界里的所有东西都是对象。

接着更多的对象都根据这个原型创造了出来：Function.prototype，Array.prototype，Date.prototype 等等。

有道是：“龙生九子不成龙，各有所好”。这些新创造的对象也不是特别像最开始的原型，而是各有所长。其中非常重要的一个就是 Function 了，它可以打包一系列操作，所以可以很方便的做一些重复的事情，比方说根据原型创建新的对象，于是它的一个很重要的角色就是构造函数。

但是既然是通过原型创造的，那么总会和原型有所关联，这个关联就是原型链，也就是__proto__。新生成的对象会继承原型上的属性和方法，这些都是天生的能力。

在对象上的属性和方法，都是沿着原型链依次向上寻找最近的那个，如果没有找到就是真的没有这个属性或方法。

比如我们创造一个猫对象 Cat，他们都继承于 Cat.prototype，然后通过 Cat 构造函数生成

        
function Cat(variety) {
  // 猫咪的品种
  this.variety = variety;
}

Cat.prototype = {
  // 猫咪都会喵喵叫
  say: function() {
    console.log('meow');
  },
  jump: function() {
    console.log('jump');
  }
};

然后我们再通过这个 Cat 原型生成两个新的对象：

        
var kangkang = new Cat('ju');
// 会捕鱼
kangkang.fishing = function() {
  console.log('fishing');
}
// 学会了神奇的叫法
kangkang.say = function() {
  console.log('wangwang');
}

var paopao = new Cat('buou');
// 会跳舞
paopao.dancing = function() {
  console.log('dancing');
}
// 学会了神奇的叫法
paopao.say = function() {
  console.log('chongya');
}

根据绿色的原型链，我们就可以知道下面的代码的执行情况了

        
kangkang.fishing(); // fishing
paopao.dancing(); // dancing

kangkang.say(); // wangwang
paopao.say(); // chongya

kangkang.jump(); // jump 沿着原型链可以找到
paopao.jump(); // jump 沿着原型链可以找到

kangkang.age // undefined 沿着原型链找到尽头也没有找到
paopao.age // undefined 沿着原型链找到尽头也没有找到
