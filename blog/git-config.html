Git Config 分为 3 个层次：系统（system），用户（global）和本地（local）。

  • 系统（system），系统配置对全部用户有效
  • 用户（global），只对该用户有效
  • 本地（local），只对该仓库有效

配置规则，以 local > global > system 覆盖配置。

配置仓库用户信息

配置用户名称和电子邮件地址。每次 Git 提交时都会引用这两条信息，说明是谁提交了更新，会随更新内容一起被永久纳入历史记录：

        
          # 公司仓库的配置
          git config --global user.name xiangtuo
          git config --global user.email xiangtuo@company.com

          # 自己仓库的配置
          git config --local user.name aximario
          git config --local user.email aximario@qq.com

查看配置

        
          git config --list
          git config user.name
          git config user.email
